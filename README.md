# LastCopy

LastCopy — локальное приложение для резервного копирования сайтов и баз данных с удалённых серверов на компьютер пользователя.

Поддерживает:
- web-режим (`localhost` в браузере),
- desktop-режим (Electron, macOS/Windows),
- выборочный бэкап (конкретные папки/файлы),
- историю бэкапов, отмену, пропуск отдельных сайтов,
- инкрементальные файлы через `rsync` (для `SSH key` / `SSH agent`).

---

## 1) Требования

- Node.js 18+ (рекомендуется 20+)
- npm 9+
- SSH-доступ к серверу
- `rsync` (для инкрементального режима файлов)

---

## 2) Установка

```bash
npm install
```

---

## 3) Запуск

### Web (локально)

```bash
npm run start
```

Открыть: `http://localhost:3000`

### Desktop (Electron)

```bash
npm run desktop
```

---

## 4) Сборка

### Web-ассеты

```bash
npm run build:web
```

### macOS

```bash
npm run build:mac
```

Результат: `dist/LastCopy-<version>-arm64.dmg`

### Windows

```bash
npm run build:win
```

Результат: установщик в `dist/` (`.exe`)

---

## 5) Первый запуск и вход

- При первом запуске у пользователя создаётся локальный конфиг.
- Если пароль ещё не задан, экран входа предложит создать пароль.
- Далее вход выполняется по этому паролю.

Смена пароля: раздел `Настройки`.

---

## 6) Настройка сервера

Раздел `Серверы` → `+ Добавить`:
- `name`, `host`, `port`, `username`,
- `authType`: `agent` / `key` / `password`,
- `basePath` (папка с сайтами на сервере).

После сохранения используйте кнопку `Тест`.

---

## 7) Создание бэкапа

Раздел `Бэкап`:
1. Выберите сервер.
2. Выберите сайты.
3. Нажмите `Продолжить`.
4. Опционально выберите конкретные папки/файлы в сайтах.
5. Выберите режим:
   - `БД + Файлы`
   - `Только БД`
   - `Только файлы`
6. Нажмите `Начать бэкап`.

Во время выполнения:
- `Остановить всё`,
- `Пропустить` отдельный сайт.

---

## 8) История

Раздел `История` показывает:
- каждый сайт отдельной строкой,
- статус (полный/неполный),
- размеры файлов и БД.

Действия по режимам:
- **Desktop (mac/win)**: кнопка `📁` открывает папку бэкапа.
- **Web**: доступны действия скачивания.

---

## 9) Настройки

В разделе `Настройки`:
- смена пароля входа,
- изменение пути хранения бэкапов (`backupDir`).

Путь проверяется на доступность для записи.

---

## 10) SSH и поведение по типу авторизации

- `authType: key` или `agent`:
  - файлы копируются через `rsync` (инкрементально),
  - БД — через `mysqldump`.

- `authType: password`:
  - файлы копируются через `tar` (без интерактивного запроса пароля в консоли),
  - БД — через `mysqldump`.

---

## 11) Где хранится конфиг

- В обычном web-режиме: `config.json` в корне проекта.
- В desktop-режиме (сборка): пользовательский конфиг хранится в папке приложения (`userData` Electron).

---

## 12) Частые проблемы

### Порт занят (`EADDRINUSE`)

- Web: освободите порт 3000 или запустите на другом.
- Desktop: приложение пытается подключиться к уже запущенному LastCopy, иначе выбирает свободный локальный порт автоматически.

### Не видно свежие изменения

Сделайте hard reload:
- macOS: `Cmd + Shift + R`
- Windows: `Ctrl + F5`

### Ошибка SSH

Проверьте:
- правильность `host/port/user`,
- тип авторизации (`agent`/`key`/`password`),
- путь `basePath`,
- доступ к серверу из вашей сети.
